> Этот файл описывает визуальный стиль, технологии и запреты.
> Любое изменение внешнего вида должно соответствовать этим правилам.

---

## 1. Стиль дизайна

### Общее направление
**Премиальный тёмный интерфейс** — dark mode, характерный для современных SaaS-продуктов уровня Linear, Vercel, Raycast.

Ощущение: дорого, чисто, профессионально. Не агрессивно, не дёшево, не шаблонно.

### Ключевые принципы

**Простота прежде всего**
Каждый экран решает одну задачу. Никакой перегруженности. Если элемент не помогает пользователю — его нет.

**Воздух и пространство**
Большие отступы между блоками. Текст не прилипает к краям. Глазу есть где отдохнуть.

**Иерархия через типографику**
Заголовок → подзаголовок → текст. Читатель понимает что важнее без объяснений.

**Акценты — редко и точно**
Фиолетовый и золотой цвета используются только там, где нужно привлечь внимание. Не везде.

**Анимации — лёгкие и функциональные**
Элементы плавно появляются при скролле. Никаких тяжёлых эффектов, которые замедляют страницу.

## 3. Типографика

### Шрифты

| Шрифт | Назначение | Начертания |
|---|---|---|
| **Manrope** | Заголовки и дисплейный текст | 600, 700, 800 |
| **Inter** | Основной текст, интерфейс | 400, 500, 600 |

Оба шрифта подключены через Google Fonts. Менять без необходимости не нужно.

### Размеры заголовков

| Уровень | Размер (мобильный) | Размер (десктоп) | Применение |
|---|---|---|---|
| H1 | 36–40px | 56–72px | Заголовок Hero |
| H2 | 28–32px | 40–48px | Заголовки секций |
| H3 | 20–22px | 22–24px | Заголовки карточек |
| Текст | 16px | 16–18px | Основной контент |
| Мелкий текст | 14px | 14px | Подписи, метаданные |

### Правила типографики
- Межстрочный интервал для текста: 1.6–1.7
- Межстрочный интервал для заголовков: 1.1–1.2
- Максимальная ширина текстового блока: 65 символов (≈ 600px)
- Заглавные буквы (UPPERCASE) — только для badge-меток и подписей категорий

## 4. Компоненты интерфейса

### Кнопки

Три вида:

**Primary** — фиолетовая, основное действие
```
Цвет фона: #7c3aed
Цвет текста: белый
Hover: #9f67ff + тень + подъём на 2px
Скругление: 12px
Отступы: 16-24px по горизонтали, 12-16px по вертикали
```

**Secondary** — прозрачная с рамкой, вторичное действие
```
Цвет фона: прозрачный
Рамка: 1px #7c3aed с прозрачностью 40%
Цвет текста: #9f67ff
Hover: фиолетовый фон 10%
```

**Ghost** — совсем тихая, третьестепенное действие
```
Цвет фона: прозрачный
Цвет текста: #8888aa
Hover: белый фон 5%
```

### Карточки

Два вида:

**Обычная карточка**
```
Фон: #13131f
Рамка: 1px #1e1e30
Скругление: 16px
Тень: внутренняя белая 5% сверху
Hover: рамка меняется на фиолетовую с прозрачностью 30%
```

**Выделенная карточка (featured)**
```
Всё то же + тень: 0 0 0 2px #7c3aed
```

### Badge (метка-бейдж)

```
Фон: цвет акцента с прозрачностью 10–15%
Рамка: 1px того же цвета с прозрачностью 20%
Скругление: 9999px (pill)
Размер текста: 12px
Отступы: 3px вертикально, 12px горизонтально
Регистр: UPPERCASE, letter-spacing: 0.05em
```

### Разделители секций

```
Тонкая линия: 1px высота
Цвет: градиент от прозрачного через акцентный к прозрачному
Пример: from-transparent via-primary/20 to-transparent
```

## 5. Сетка и отступы

### Контейнер
- Максимальная ширина: `1280px` (max-w-7xl)
- Горизонтальные отступы: 16px (мобильный) → 24px (планшет) → 32px (десктоп)

### Отступы секций
- Вертикальный padding каждой секции: `96px` сверху и снизу
- На мобильных: `64px`

### Сетка карточек
- 1 колонка (мобильный < 640px)
- 2 колонки (планшет 640–1024px)
- 3 колонки (десктоп > 1024px)
- Промежуток между карточками: 16–20px

## 6. Анимации

### Принцип
Анимации помогают — не развлекают. Каждая анимация имеет причину.

### Типы анимаций

**Появление при скролле (основная)**
```
Эффект: элемент появляется снизу вверх + плавное нарастание прозрачности
Длительность: 500–700ms
Задержка между карточками: 60–100ms (чтобы появлялись по очереди)
Триггер: элемент вошёл в зону видимости на 10%
```

**Hover на кнопках**
```
Подъём: translateY(-2px)
Длительность: 200–300ms
```

**Hover на карточках**
```
Изменение рамки и фона
Длительность: 300ms
```

**Пульсация (glow)**
```
Использовать редко: только на ключевом CTA или иконке
Цикл: 3 секунды
```

### Что запрещено в анимациях
- ❌ Автоматические слайдеры (carousel) — дезориентируют
- ❌ Параллакс-эффекты — тяжело для слабых устройств
- ❌ Анимации дольше 1 секунды — раздражают
- ❌ Анимации, которые не остановить — нарушают доступность

## 7. Адаптивность (responsive design)

### Контрольные точки

| Название | Ширина | Устройство |
|---|---|---|
| Мобильный | < 640px | Телефон |
| Планшет | 640–1024px | Планшет, большой телефон |
| Десктоп | > 1024px | Ноутбук, монитор |

### Правила адаптивности
- Сначала проектируем под мобильный, потом расширяем
- Кнопки на мобильном — минимум 44×44px (правило Apple HIG)
- Текст на мобильном — минимум 16px (чтобы браузер не зумировал)
- Горизонтальная прокрутка — запрещена
- Каждое изменение проверяем на телефоне

## 8. Технологии

### Текущий стек (лендинг)

| Инструмент | Версия | Зачем |
|---|---|---|
| **React** | 18 | Компонентный UI — каждая секция отдельный файл |
| **Vite** | 6 | Быстрая сборка, локальный сервер |
| **Tailwind CSS** | 3 | Стили прямо в компоненте, без отдельных CSS-файлов |
| **Lucide React** | последняя | Иконки — чистые, современные, единообразные |

### Почему именно этот стек
- Tailwind — стандарт для быстрой разработки без дизайнера
- React — легко переиспользовать компоненты
- Vite — самый быстрый инструмент сборки в 2025–2026
- Lucide — одна библиотека иконок для всего проекта

### Стек для будущей платформы (планируется)
- **Next.js** — фронтенд с серверным рендерингом (для SEO и скорости)
- **FastAPI** (Python) — бэкенд
- **PostgreSQL** — база данных
- **aiogram 3** — Telegram-бот
- **ЮКасса** — приём платежей

## 9. Что нельзя делать — запреты

### Дизайн
- ❌ Белый или светлый фон — проект построен на тёмной теме
- ❌ Яркие кислотные цвета вне палитры
- ❌ Более 2 акцентных цветов в одном блоке
- ❌ Шрифты кроме Inter и Manrope
- ❌ Текст поверх сложного фото без подложки
- ❌ Маленькие кнопки — минимум 44px по высоте
- ❌ Текст меньше 14px
- ❌ Горизонтальный скролл

### Контент
- ❌ Стоковые фотографии людей (выглядят неестественно)
- ❌ Клипарт и мультяшные иллюстрации — не соответствуют стилю
- ❌ Lorem ipsum и любые заглушки в опубликованной версии
- ❌ Текст полностью ЗАГЛАВНЫМИ БУКВАМИ в основном контенте
- ❌ Более 3 уровней вложенности в тексте

### Технологии
- ❌ Подключать новые библиотеки без обсуждения
- ❌ Менять версии зависимостей руками в package.json
- ❌ Хранить пароли и ключи в коде
- ❌ Использовать inline-стили вместо Tailwind
- ❌ Писать CSS в отдельных файлах (кроме index.css)

### Публикация
- ❌ Публиковать непроверенную версию
- ❌ Вносить правки напрямую на Netlify, минуя Git
- ❌ Удалять файлы из репозитория без обсуждения
